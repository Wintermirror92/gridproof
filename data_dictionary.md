# GridProof ESS 데이터 점검 결과

## 1. 공개데이터: MVP에서의 역할
공개데이터는 실제 사업지 이행검증용 원시 로그가 아니라, ESS 검증 필요성의 배경과 지역별 환경 설명에 사용한다.

## 2. 확인된 주요 데이터셋과 컬럼

### 한국전력거래소_전력시장 참여 태양광 발전설비 전력거래량(AI 친화데이터)
- 행 수: 148,920
- 기간: 2025-01-01 ~ 2025-12-31
- 컬럼: `trade_date`, `trade_hour`, `region_code`, `fuel_type`, `trade_volume`
- 용도: 전남 태양광 전력거래량 시간대별 패턴, 재생에너지 변동성 설명

### 한국전력거래소_신재생연계ESS 설비용량 자료
- 최신 파일 기준: 2025년 5월
- 컬럼: `연도`, `월`, `지역`, `태양광`, `풍력`, `전기저장장치(태양광)`, `전기저장장치(풍력)`
- 파생 컬럼: `ESS연계설비용량_MW = 전기저장장치(태양광) + 전기저장장치(풍력)`
- 용도: 전남 ESS 연계 설비용량 및 지역별 비교

### 전국 태양광·풍력 출력제어횟수 및 제주 풍력 제어량
- 컬럼: `일자`, `제어지역`, `누적 시행회차`, `1시`~`24시`, 일부 파일은 `총제어량`
- 값 형태: 육지 출력제어 파일은 시간대 값이 `출력제어`/`0` 혼합, 제주 풍력 파일은 시간대별 제어량 수치 포함
- 용도: 출력제어 발생일·시간대 맥락 설명

### 한국전력거래소_지역별 시간별 계량데이터
- 연도별 파일: 2013~2025
- 컬럼: `거래일`, `거래시간`, `지역`, `계량값[GWh]`
- 용도: 지역별 시간대 전력 계량 추이 설명. ESS 사업지 이행검증 원시 로그는 아님.

### 한국전력거래소_지역별 시간대별 전력거래량
- 컬럼: `거래일`, `거래시간`, `지역`, `전력거래량`
- 용도: 지역별 전력거래량 맥락 설명

### 한국전력거래소_지역별 시간별 태양광 및 풍력 발전량
- 컬럼: `거래일`, `거래시간`, `지역`, `연료원`, `전력거래량(MWh)`
- 용도: 재생에너지 시간대별 발전량 맥락 설명

### 한국전력거래소_지역별 5분단위 태양광/풍력 계량데이터
- 컬럼: `연료원`, `지역`, `거래일`, `시간`, `0분`, `5분`, ... `55분`
- 용도: 시간 해상도가 높은 재생에너지 변동성 설명. ESS 개별 지령 이행 여부는 직접 판정할 수 없음.

## 3. 합성데이터 스키마

### ess_sites.csv
`site_id`, `site_name`, `municipality`, `contract_kw`, `contract_kwh`, `installed_kw`, `installed_kwh`, `gps_match`, `doc_missing`, `operator`, `lat`, `lon`, `profile`

### hourly_verification_log.csv
`timestamp`, `site_id`, `site_name`, `municipality`, `command_kw`, `actual_kw`, `fulfillment_rate`, `soc_pct`, `comm_status`, `pcs_alarm`, `bms_alarm`, `quality_flag`, `weather_alert`, `response_delay_min`, `low_fulfillment`, `soc_low`, `data_type`

### site_summary.csv
사업지별 평균 이행률, 통신 결측률, SoC 부족률, 알람률, 품질플래그율, 위험점수, 주요 단서, 권고 조치.

## 4. 팩트체크
- 실제 파일 컬럼 기준으로 확인한 결과, 공개데이터에는 `site_id`, `KPX dispatch command`, `ESS SoC`, `BMS/PCS alarm`, `quality flag`가 없다.
- 따라서 MVP 설명은 “실제 데이터 기반 ESS 이행검증 완료”가 아니라 “공개데이터 기반 환경 분석 + 합성 로그 기반 End-to-End 이행검증 프로세스 구현”이어야 한다.
